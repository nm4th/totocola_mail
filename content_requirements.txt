## 概要

毎週、4本のメルマガHTMLファイルを自動生成し、
デスクトップの「ととコーラメルマガ」フォルダに格納する。
配信は月・火・木・土の週4本。
担当者が月曜日に手動でShopify Emailにインポートして配信する。

----------------------------------------------------------------

## 実行タイミング・出力先

- 毎週日曜日の夜〜月曜日の朝までに自動実行
- 出力先：~/Desktop/ととコーラメルマガ/YYYY-MM-DD/（実行週月曜日のフォルダ）

----------------------------------------------------------------

## ファイル名ルール

形式：YYYY-MM-DD_曜日_件名.html

例：
  2026-04-27_月_先のことを考えすぎて、今が楽しめない理由.html
  2026-04-28_火_眠いのに、なぜかスマホが止まらない夜があります。.html
  2026-04-30_木_完璧にしようとするほど、動けなくなる理由.html
  2026-05-02_土_久しぶりに、何もしない土曜日を過ごしました。.html

- 日付は配信予定日（月曜から月・火・木・土で割り当て）
- ファイル名に使えない文字（/ \ : * ? " < > |）は除去する

----------------------------------------------------------------

## 配信曜日と日付の割り当て

月曜日 = 実行週の月曜日
火曜日 = 実行週の火曜日
木曜日 = 実行週の木曜日
土曜日 = 実行週の土曜日

----------------------------------------------------------------

## Claude API連携仕様

毎週4本の本文をClaude APIで生成する。
既存のメルマガはトーン・構造の参考例として使用する。

モデル    : claude-sonnet-4-6
max_tokens: 2000
形式      : JSONのみ返すよう指示（```なし）
リクエスト: 1本ずつ4回（品質安定のため）

## 件名の型と検証ルール

データ検証の結果、以下の型とルールを採用する。

【件名の型】毎週3種類をバランスよく使う
  A. 「〜理由」型  ：「好きなことが、わからなくなってきた理由」
  B. 断言型        ：「眠いのに、なぜかスマホが止まらない夜があります。」
  C. 疑問形        ：「昔より、感動しにくくなったのはなぜでしょうか？」

【件名の長さ】
  短め（〜20文字）が高開封率の傾向あり
  長め（21文字〜）はぼんやりした印象になりやすい

【あるある感】
  具体的なシーン・言葉を入れる（「スマホ」「土曜日」「また今度」など）
  抽象的すぎる件名は開封率が下がる傾向あり

【過去の高開封率件名（参考）】
  - 「また今度」が続く関係が、少しずつ遠くなる理由  44.3%
  - タイパを求めるほど、満たされなくなる理由         43.41%
  - 頑張れるのに、続かない理由                       42.53%
  - 早く過ぎる毎日は、悪いことなのでしょうか？       42.59%
  - 何もしたくない日が続く理由                       41.32%

----------------------------------------------------------------

## プレビューテキストの型

件名の型と同様に毎週混ぜて検証する。

  A. 答えを匂わせる：「今この瞬間を楽しめないのは、考えすぎているからかもしれません。」
  B. 共感を重ねる  ：「横になっても、手が動いてしまう。」
  C. 問いかけで終わる：「あなただけじゃないかもしれません。」

※検証結果：AとBが高開封率の傾向。Cは効果が弱い可能性あり。
※Cは引き続き少量で検証を継続する。

【テーマの方向性】
  - 人間関係・他者との距離系
  - 自分の内側・感情系
  - 夜の行動・習慣系
  ※毎週バランスよく混ぜる。同じ方向性が続くと開封率が下がる傾向あり。

【テーマの具体性】

## Claude APIへのプロンプト仕様

--- system ---
あなたはととコーラというブランドのメルマガライターです。
以下の要件定義に従って、メルマガ本文をJSON形式で生成してください。

【ブランド情報】
- 商品: ととコーラ（ノンカフェイン・低カロリー・無添加のスパイスシロップ）
- コンセプト: 「前に進む前に、立ち止まる」
- ポジション: 「立ち止まるための飲み物」
- ターゲット: 購入経験のある読者（夜に自分の時間を大切にしたい30代前後）

【文体の原則】
- 読者目線で語りかける（送り手目線にならない）
- 断定せず「〜かもしれません」「〜と思います」
- 説教しない。共感から始めて、そっと視点を渡す
- 夜の静けさに合うトーン。押しつけがましくない
- 短い文を積み重ねるリズム感
- 難しい言葉を使わない

【構造】
- 冒頭ブロック: 読者が「自分のことだ」と感じる情景描写 + 問いかけ（100〜150文字）
- セクション3〜4つ: 見出し + 共感 + 視点転換 + ととコーラへの接続
- 締めの一文 + 🌙 ととコーラ でサインオフ

【禁止事項】
- 「！」の多用（1通に1回まで）
- 「〜しましょう」
- 商品スペックの列挙
- 送り手（ととコーラ）の日記・体験談にならないこと
- ととコーラの名前は本文中1〜2回まで

必ずJSONのみ返してください。```や説明文は不要です。

--- user ---
以下の条件でメルマガを1本生成してください。

件名の型: {type} （reason=〜理由型 / statement=断言型 / question=疑問形）
プレビューの型: {preview_type} （A=答えを匂わせる / B=共感を重ねる / C=問いかけで終わる）
テーマの方向性: {theme_direction}
件名の長さ: {length} （short=20文字以内 / long=21文字以上）
使用済みテーマ（重複禁止）: {used_themes}

以下のJSON形式で返してください:
{
  "subject": "件名",
  "preview": "プレビューテキスト（25〜40文字）",
  "subtitle": "サブタイトル（ヘッダー下に表示・10文字前後）",
  "theme": "今回使用したテーマ（used_themesに追記するため）",
  "opening": {
    "paragraphs": ["冒頭の文章1", "冒頭の文章2", "冒頭の文章3（問いかけ）"]
  },
  "sections": [
    {
      "heading": "見出し（10文字前後）",
      "paragraphs": ["段落1", "段落2"],
      "bullets": [],
      "emphasis": "この段落内で強調する一文（なければ空文字）"
    }
  ],
  "closing": {
    "paragraphs": ["締めの文章1", "締めの文章2"]
  }
}

bulletsは羅列が3つ以上ある場合のみ使用。不要な場合は空配列[]。
emphasisは各セクションで最も伝えたい一文を指定。なければ空文字。

## SVGイラストのルール（次フェーズで実装予定）

各セクション見出しの直後にSVGイラストを1つ入れる。

【基本ルール】
- 描く対象は1つだけに絞る（要素を入れすぎない）
- 見た瞬間に何かわかる具体的なもの
- 人体・顔など複雑な形は描かない
- カラーはゴールド（#c9a84c）とグレー（#8b92a8）と暗色（#1e2436 / #141824）のみ使用

【描ける対象の例】
  脳、砂時計、グラス（コップ）、スマホ、
  夜空＋三日月＋家のシルエット、窓＋カーテン、
  コーヒーカップ（湯気あり）、開いた本、
  鍵穴、天秤、足跡、目

【描けない対象（使用禁止）】
  人物・顔・手足のみ（体全体が難しい）
  複雑な風景、動物、文字を含むもの

【夜空＋家のシルエット（締めに使う定番イラスト）】
  三日月＋星＋地平線＋家（屋根＋窓に灯り）
  窓の灯りで「今夜ここにいる」を表現
  締めのセクションに使うことが多い
